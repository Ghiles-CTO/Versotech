# Arranger Guide: Requesting Deal Close

Submit a close request when your deal is ready to finalize.

---

## Overview

Closing a deal transitions all funded subscriptions to active positions. This requires CEO/staff approval and triggers several automated processes.

---

## When to Request Close

### Prerequisites Checklist

Before requesting close, verify:

| Requirement | Status |
|-------------|--------|
| Target raise achieved (or acceptable) | ✅ |
| All subscriptions funded | ✅ |
| All documents signed | ✅ |
| KYC complete for all investors | ✅ |
| No outstanding compliance issues | ✅ |

### Partial Close Considerations

For deals accepting multiple closes:
- First close: Minimum viable amount
- Subsequent closes: Additional investors
- Final close: No more subscriptions

---

## Pre-Close Review

### Step 1: Review Subscriptions

1. Go to deal **Subscriptions** tab
2. Verify all are `funded` status
3. Check for any exceptions

### Step 2: Check Documents

1. Go to deal **Documents** tab
2. Ensure all required docs present
3. Verify signatures complete

### Step 3: Review Fee Plans

1. Go to deal **Fee Plans** tab
2. Confirm all fee plans accepted
3. Commission calculations ready

---

## Submitting Close Request

### Step 1: Access Close Request

1. Go to deal **Overview**
2. Click **"Request Close"**
3. Close request form opens

### Step 2: Complete Request Form

| Field | Description |
|-------|-------------|
| **Close Date** | Proposed closing date |
| **Close Type** | First, Interim, Final |
| **Subscriptions** | Which subs to include |
| **Notes** | Any special instructions |

### Step 3: Review Summary

The system shows:
- Total amount closing
- Number of investors
- Commission liability
- Any warnings

### Step 4: Submit Request

1. Confirm all details correct
2. Click **"Submit Close Request"**
3. Request goes to CEO/staff queue
4. Status shows as `pending_close`

---

## What Happens After Submission

### Approval Workflow

```
Close Request Submitted
    → Staff Review
        → CEO Approval
            → Close Executed
```

### Staff Review

Staff verifies:
- All subscriptions properly funded
- Documents in order
- Compliance checks passed
- No outstanding issues

### CEO Approval

CEO confirms:
- Deal meets requirements
- Ready to formalize
- Approves the close

---

## Close Execution

### When Approved

Automated processes trigger:

1. **Subscriptions Updated**
   - Status changes to `active`

2. **Positions Created**
   - Each investor gets position record
   - Units allocated
   - Cost basis recorded

3. **Certificates Generated**
   - Share/unit certificates created
   - Sent to investors

4. **Commissions Triggered**
   - Move to `accrued` status
   - Ready for invoice process

5. **Notifications Sent**
   - Investors notified of close
   - Certificates distributed
   - Welcome communications

## Tracking Close Status

### Close Request States

| Status | Meaning |
|--------|---------|
| `submitted` | Awaiting review |
| `under_review` | Being reviewed |
| `approved` | Ready to execute |
| `executed` | Close complete |
| `rejected` | Not approved |

### Viewing Status

1. Go to deal **Overview**
2. See close request status
3. Click for details

---

## Handling Rejection

### If Close Request Rejected

1. Review rejection reason
2. Address the issues
3. Resubmit when resolved

### Common Rejection Reasons

| Reason | Resolution |
|--------|------------|
| Unfunded subscriptions | Wait for funding or exclude |
| Missing documents | Upload required docs |
| KYC incomplete | Complete investor KYC |
| Compliance issue | Resolve with compliance |

## Post-Close Actions

### Immediate Actions

After close is executed:
1. Verify all positions created
2. Confirm certificates generated
3. Review commission accruals

### Investor Communications

Standard communications sent:
- Close confirmation
- Certificate delivery
- Next steps/welcome

### Commission Processing

Commissions now ready:
- Status is `accrued`
- Can request invoices from introducers
- See [Processing Commissions](./11-processing-commissions.md)

## Multiple Closes

### First Close

Initial closing:
- Minimum viable amount
- Core investors
- Deal becomes active

### Interim Closes

Additional closings:
- Add more investors
- Same process as first close
- Pro-rata or full allocation

### Final Close

Last closing:
- No more subscriptions accepted
- Deal status changes to `closed`
- Full commission reconciliation

## Common Issues

### "Cannot request close - unfunded subscriptions"

Options:
1. Wait for funding to complete
2. Exclude unfunded from this close
3. Convert to partial close

### "Close request stuck in review"

1. Check if additional info needed
2. Contact reviewing staff
3. Escalate if urgent

### "Positions not created after close"

1. Verify close was executed
2. Check for system issues
3. Contact VERSO support

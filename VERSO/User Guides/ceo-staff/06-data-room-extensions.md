# CEO & Staff Guide: Data Room Extensions

Handle investor requests for extended data room access.

---

## Overview

When investors need more time to review data room materials beyond the standard 7-day window, they can request an extension. Staff approves or denies these requests.

---

## Why Extensions Are Requested

### Common Reasons

| Reason | Validity |
|--------|----------|
| **More due diligence needed** | Usually valid |
| **Advisor review pending** | Usually valid |
| **Travel/unavailability** | Usually valid |
| **Decision delay** | May or may not be valid |

## Reviewing Requests

### Step 1: Open Request

Click to see full details.

### Step 2: Review Context

Consider:
| Factor | Question |
|--------|----------|
| **Reason** | Is it legitimate? |
| **History** | First request or repeat? |
| **Engagement** | Are they actively reviewing? |
| **Timeline** | Does deal timeline permit? |

### Step 3: Check Activity

Review investor's data room activity:
- Documents viewed
- Time spent
- Last access
- Download patterns

---

## Making a Decision

### Approving Extension

If reasonable request:
1. Click **"Approve"**
2. Set new expiry date
3. Optionally add conditions
4. Investor notified
5. Access extended

### Denying Extension

If not appropriate:
1. Click **"Deny"**
2. Enter reason
3. Investor notified
4. Access expires as scheduled

---

## Extension Duration

### Standard Extensions

| Scenario | Typical Extension |
|----------|-------------------|
| **First request** | 7 additional days |
| **Second request** | 3-7 days |
| **Complex deal** | Case by case |

### Maximum Extensions

Set reasonable limits:
- Consider deal timeline
- Fairness to other investors
- Document management

---

## Approval Considerations

### Grant When

- Investor engaged but needs more time
- Legitimate scheduling issues
- Complex documentation
- Deal timeline permits

### Deny When

- Repeated requests without progress
- Deal closing soon
- No engagement visible
- Security concerns

---

## Conditional Approvals

### Adding Conditions

May approve with:
- Specific deadline
- Expectation of decision
- Required engagement

### Communication

Clear expectations:
- State the new expiry
- Note any conditions
- Encourage timely review

## Best Practices

### Consistency

Apply similar standards:
- Similar situations = similar decisions
- Document rationale
- Fair treatment

### Communication

Clear responses:
- Approve with specific dates
- Deny with explanation
- Encourage next steps

### Balance

Consider:
- Investor needs
- Deal timeline
- Fairness
- Security

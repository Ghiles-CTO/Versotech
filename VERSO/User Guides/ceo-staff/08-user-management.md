# CEO & Staff Guide: User Management

Create and manage user accounts on the platform.

---

## Overview

User management includes creating accounts, assigning personas, managing access, and handling user issues.

---

## Accessing User Management

### Navigation

1. Go to **Users** in left sidebar
2. See all platform users
3. Filter and search options

### User List

Displays:
- User name
- Email
- Personas
- Status
- Last active

---

## Creating Users

### Step 1: Start Creation

1. Click **"+ Add User"**
2. Creation form opens

### Step 2: Basic Information

| Field | Required |
|-------|----------|
| **Email** | Yes |
| **First Name** | Yes |
| **Last Name** | Yes |
| **Phone** | Recommended |

### Step 3: Assign Personas

Select applicable personas:
- [ ] Investor
- [ ] Arranger
- [ ] Introducer
- [ ] Partner
- [ ] Commercial Partner
- [ ] Lawyer
- [ ] Staff

Multiple personas can be assigned.

### Step 4: Send Invitation

1. Review details
2. Click **"Create & Invite"**
3. User receives email invitation
4. Status: `pending`

---

## User Statuses

| Status | Meaning |
|--------|---------|
| `pending` | Invited, not yet activated |
| `active` | Account in use |
| `suspended` | Temporarily disabled |
| `deactivated` | Permanently disabled |

---

## Managing Existing Users

### Viewing User Details

Click on any user to see:
- Profile information
- Assigned personas
- Activity history
- Linked entities

### Editing User

1. Click **"Edit"**
2. Modify fields
3. Save changes

### Persona Changes

To add/remove personas:
1. Open user profile
2. Go to **Personas** section
3. Toggle personas
4. Save

---

## User Actions

### Resend Invitation

For pending users:
1. Click **"Resend Invitation"**
2. New email sent
3. Previous link invalidated

### Reset Password

If user needs reset:
1. Click **"Reset Password"**
2. User receives reset email
3. They create new password

### Suspend User

Temporarily disable access:
1. Click **"Suspend"**
2. Confirm action
3. User cannot log in
4. Can be reactivated

### Deactivate User

Permanently disable:
1. Click **"Deactivate"**
2. Confirm action
3. Account disabled
4. Data retained for audit

---

## User Permissions

### Persona-Based Access

Access determined by personas:
- Each persona has defined capabilities
- Multiple personas = combined access
- Persona can be removed

### Special Permissions

Some users may need:
- Additional approval authority
- Specific feature access
- Custom permissions

---

## Bulk Operations

### Importing Users

For multiple users:
1. Click **"Import"**
2. Download template
3. Fill in user data
4. Upload file
5. Review and confirm

### Exporting User List

1. Click **"Export"**
2. Select format
3. Download file

---

## User Issues

### Locked Accounts

If user is locked:
1. Check login attempts
2. May need password reset
3. Or manual unlock

### Access Problems

If user can't access:
1. Verify account status
2. Check persona assignments
3. Confirm email correct

### Multiple Accounts

If user has duplicates:
1. Identify primary account
2. Merge or deactivate extras
3. Update references

---

## Audit Trail

### User Activity

For each user:
- Login history
- Actions taken
- Changes made

### Access History

Track:
- When created
- Persona changes
- Status changes

---

## Best Practices

### Account Creation

- Verify email addresses
- Assign appropriate personas
- Document the purpose

### Regular Review

- Audit user list periodically
- Remove unused accounts
- Update personas as needed

### Security

- Prompt password resets if needed
- Suspend suspicious accounts
- Monitor for anomalies
